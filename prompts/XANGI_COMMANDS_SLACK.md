# XANGI_COMMANDS_SLACK.md - Slack専用コマンド

## Slack操作コマンド

**⚠️ `!slack` コマンドはBashコマンドではない！**
応答テキストに直接書くこと。Bash ツールで実行すると `command not found` エラーになる。
xangi がテキスト出力を各行ごとにパースして処理する仕組み。

### 別チャンネルにメッセージ送信

```
!slack send <#チャンネルID> メッセージ内容
!slack send <#チャンネルID> thread:スレッドts メッセージ内容
```

**例：**
```
!slack send <#C1234567890> hello!
!slack send <#C1234567890> 作業開始します
!slack send <#C1234567890> thread:1234567890.123456 スレッド内に返信
```

**注意：**
- `<#チャンネルID>` の形式を守ること（`<#` と `>` で囲む）
- 「〇〇チャンネルに△△って言って」と頼まれたら、このコマンドを使う
- `thread:ts` を付けるとスレッド内に送信できる
- 複数行メッセージに対応（次の `!slack` / `!schedule` コマンド行まで吸収）

### チャンネル一覧を取得

```
!slack channels
```

bot が参加しているチャンネルの一覧を返す。**結果はSlackに送信されず、自分のコンテキストに返る。**

### チャンネル履歴の取得

```
!slack history [件数] [offset:N] [<#チャンネルID>]
```

チャンネルの最新メッセージを取得する。**結果はSlackに送信されず、自分のコンテキストに返る。**

- 件数省略時はデフォルト 10 件、最大 100 件
- チャンネル ID 省略時は現在のチャンネル
- `offset:N` で古いメッセージに遡れる（30 件ずつ取得でタイムアウト防止）

**例：**
```
!slack history              # 現在のチャンネル最新10件
!slack history 50           # 現在のチャンネル最新50件
!slack history 20 <#C1234567890>  # 指定チャンネル20件
!slack history 30 offset:30   # 30〜60件目を取得
!slack history 30 offset:60   # 60〜90件目を取得
!slack history 30 offset:30 <#C1234567890>  # 別チャンネルで遡る
```

**大量の履歴を取得したい場合（タイムアウト防止）：**
100 件一括取得ではなく、30 件ずつ offset で遡ること：
1. `!slack history 30` → 最新 30 件
2. `!slack history 30 offset:30` → 30〜60 件目
3. `!slack history 30 offset:60` → 60〜90 件目

**用途：**
- セッションリセット後に会話の文脈を把握
- 過去の会話を参照して判断材料にする

### メッセージ検索

```
!slack search キーワード
```

現在のチャンネルで最新 100 件からキーワードを含むメッセージを検索する。**結果はSlackに送信されず、自分のコンテキストに返る。**

### メッセージ削除

```
!slack delete                  # 直前のBotメッセージを削除
!slack delete <ts>             # 指定tsのメッセージを削除
!slack delete <メッセージリンク>  # リンク先メッセージを削除
```

- ts（タイムスタンプ）またはメッセージリンクでメッセージを指定
- 引数なしの場合は直前の Bot メッセージを削除

### チャンネルトピックの変更

```
!slack topic <#チャンネルID> 新しいトピック
```

**例：**
```
!slack topic <#C1234567890> 今週のスプリント目標
```

**注意：**
- bot に適切な権限が必要

## ファイル送信

チャットにファイルを送信する場合は、出力に以下の形式でパスを含める（**行頭でなくてもOK**、テキスト途中でも認識される）：

```
MEDIA:/path/to/file
```

**対応形式:** png, jpg, jpeg, gif, webp, mp3, mp4, wav, flac, pdf, zip

**例：**
```
画像を生成しました。
MEDIA:/tmp/output.png
```

ユーザーが添付したファイルは `[添付ファイル]` としてパスが渡される。
